## 0. 원칙

- 토큰 0: 문제·해설 LLM 생성 금지. 추출 → 규칙분류 → 발행만.
- 로그인은 개인화 기능(오답노트·다시풀기·신고)에만. 풀이·정리 페이지는 익명 허용.
- 인증은 구글 OAuth 단일. 이메일 가입·비밀번호 관리 없음.

## 1-1. MVP 범위

**MVP = M0~M2**: F1 파이프라인 + F2 풀이·채점 + F3 유형 페이지 + F4 허브 + F10 SEO + F11 셔플.
로그인 없이 돌아가는 전부. 여기까지 배포하고 색인 상태 보고 M3(로그인·오답노트·신고) 진행.

- F10: MVP 포함 — 색인 안 되면 수익 동선 자체가 없음.
- F11: MVP 포함 — 구현 반나절, comcbt 원문과 텍스트 차별화 부수 효과.
- F12: post-MVP — 트래픽 확인 후.

## 1-2. 데이터 현황 (2026-07-21)

- 3과목 × 4회차(122~125회) 파싱: **301문항 적재** (이론 179 + 실무분개 76 + 결산 46, 중복 stem 제거)
- 파싱 성공률 302/303 (실패 1건은 `out/failed.json` 보존, `--retry-failed`로 재처리)
- type_tag 26종, 분류율 81% (미분류 56건 — 규칙 추가로 지속 개선)
- 세무2급 가로 2단 조판 대응(페이지 방향 감지), 정답표는 원시 텍스트에서 추출
- DB 뷰 `question_tag_counts` — PostgREST 1000행 캡 회피용 집계

## 1-3. 안정성 (에이전트 리뷰 반영, 2026-07-21)

- Supabase 조회 3회 재시도(지수 백오프) — 빌드 타임 일시 522에도 빌드 생존
- 사이드바 nav 실패 시 빈 배열 폴백 (DB 장애가 사이트 전체를 죽이지 않음)
- 유형 페이지 `dynamicParams=true` + 0건 `notFound()` — 새 유형 재배포 없이 생성
- 퀴즈/오답노트 에러 상태 + 다시 시도 버튼, 신고 버튼 중복 제출 방지
- 로그인 기록 실패 시 localStorage 폴백, 이관 시 FK 불량 행 개별 처리
- 접근성: muted 대비 AA 충족, focus-visible 링, prefers-reduced-motion, 모바일 오버레이 Esc 닫기
- 키보드 단축키: 퀴즈에서 1~4 보기 선택, Enter 다음 문제 (F12에서 조기 구현)

## 2. 기능 상세

### F10. SEO 기본 세트 (MVP)

- `sitemap.xml` 자동 생성 (Next.js `app/sitemap.ts` — F3 전 조합 + 허브).
- 페이지별 title/description/OG 메타 (유형명·문항수 포함, 템플릿 기반).
- F3 페이지 하단 "관련 유형" 내부 링크 4~6개 (같은 subject 내 다른 type_tag).

### F11. 보기 순서 셔플 (MVP)

- `/quiz` 렌더 시 보기 4개 순서 랜덤, 채점은 원본 answer_idx 매핑으로 판정.
- F3 페이지는 빌드 타임 셔플 1회 고정 (SSG라 요청마다 못 섞음, comcbt 원문과 순서 달라지는 효과는 동일).

### F12. 이어풀기 + 잔챙이 (post-MVP)

- 진행 상태 localStorage 저장 → 재방문 시 "이어서 풀기".
- 키보드 단축키(1~4·Enter), 다크모드, 시험 D-day 배너.
- 트래픽 확인 후 착수. 상세 정의는 그때.

### F1. 기출 파싱 파이프라인 (오프라인, Python) — ✅ 검증 완료 (2026-07-21)

- comcbt 게시판에서 회차 zip 자동 다운로드 (`fetch_comcbt.py`). `raw/` 캐시, 재실행 시 스킵.
- 소스는 zip 안의 **세무사회 공식 확정답안 PDF** (hwp 아님) — 문제·보기·정답·해설 전부 포함, 이 파일 하나만 파싱.
- `parse_questions.py` (PyMuPDF): 좌표 기반 파싱(줄바꿈 보기 병합), 상단 A형 정답표와 [답] 교차검증, 복수정답 인정 처리.
- **이론** 15문항/회차: 4지선다 + 해설. **실무**는 분개(일반전표입력)·결산만 추출 —
  프로그램 조작형(기초정보·매입매출전표·오류수정·조회)은 제외. 분개·결산은
  `answer_text`(정답 분개)로 저장, 4지선다 보기 생성은 별도 단계에서 판단.
- 출력: `out/questions.json`, 실패분 `out/failed.json` (원문+사유), `--retry-failed`로 재파싱.
- type_tag 키워드 규칙 분류(분개는 stem+답 기준). 미매칭 `미분류`.
- 검증 결과: 전산회계1급 122~125회 4개 회차, 이론 60/60 + 분개 24 + 결산 12 = 96건 실패 0, 해설 100%.

### F2. 4지선다 풀이·자동 채점 (`/quiz`)

- 필터(과목·category·type_tag)로 문제 세트 로드, 한 문제씩 진행.
- 보기 선택 → 즉시 채점(`chosen_idx === answer_idx`) → 정답·해설 표시.
- 기록: 익명은 localStorage, 로그인 유저는 `attempts` insert.
- 로그인 시 localStorage 기록을 attempts로 일괄 이관 후 비움.
- 결과 화면에 "오답 저장하려면 로그인" 유도.

### F3. 유형별 정리 페이지 (`/[subject]/[type_tag]`, SSG)

- 빌드 타임에 (subject, type_tag) 전 조합 정적 생성. **구글 유입 + 애드센스 = 수익 동선.**
- 문제 stem·보기는 서버 렌더(색인용), 정답·해설은 클릭 시 공개(인터랙티브 차별화).
- 원문 통짜 나열 금지 — 유형 재구성 + 툴 형태 유지.

### F4. 색인 허브 (`/`, SSG)

- 과목 → 유형 링크 트리. F3 페이지 전부로 내부 링크. 크롤러 진입점.

### F5. 구글 로그인

- Supabase Auth 구글 OAuth 단일. `/auth/callback` 라우트 하나.
- 로그인 버튼 노출 위치: `/wrong` 진입 시, `/quiz` 결과 화면.
- 미들웨어 없음 — 보호 라우트가 적어 페이지 내 세션 체크로 처리.

### F6. 오답노트 (`/wrong`)

- `attempts where is_correct = false` + questions 조인, 최근순.
- 같은 문제 여러 번 틀리면 최신 시도 기준 표시(중복 제거).
- RLS로 본인 행만 조회.

### F7. 오답 다시풀기

- `/wrong` 상단 "다시 풀기" 버튼 → `/quiz?mode=wrong`.
- QuizRunner 재사용, 문제 소스만 오답 목록으로 교체.
- 다시 맞히면 새 attempts 행(정답)이 쌓여 오답노트에서 자연 제외.

### F8. 문제 신고

- 문제 카드에 "신고" 버튼 → 사유 선택(정답 오류 / 보기 깨짐 / 해설 오류 / 기타) + 선택 메모.
- `reports` 테이블 insert. 로그인 유저만(스팸 방지).
- 처리 UI 없음 — 운영자가 Supabase 대시보드에서 직접 확인·수정. 파싱 품질 피드백 루프.

### F9. 애드센스

- F3 페이지에 광고 슬롯. 레이아웃에 스크립트 1회 삽입.
- 승인 신청은 유형 페이지 수십 개 색인 후.

## 3. DB 추가분 (기능정의로 생긴 변경)

기존 `questions`, `attempts`에 추가:

```sql
-- F8 문제 신고
reports (
  id           uuid primary key default gen_random_uuid(),
  user_id      uuid not null references auth.users on delete cascade,
  question_id  uuid not null references questions on delete cascade,
  reason       text not null,   -- 정답오류 | 보기깨짐 | 해설오류 | 기타
  memo         text,
  created_at   timestamptz default now()
);
-- RLS: insert만 본인, select는 없음(운영자는 대시보드에서 service_role로 봄)
```

F7은 스키마 변경 없음 (attempts 재사용).

## 4. 안 하는 것 (보류 확정)

- 이메일+비밀번호 가입, 카카오 로그인
- 검색, 북마크, 스트릭/게이미피케이션, 통계 대시보드
- 문제별 개별 페이지(permalink) — comcbt 중복으로 SEO 자해
- 회차 모의고사 모드 — 트래픽 생긴 후 재검토
